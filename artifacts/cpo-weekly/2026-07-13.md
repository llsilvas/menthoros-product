## 7. Fora da caixa

### 7.1 Dado já coletado, valor não extraído

O Menthoros já coleta dados que nenhum concorrente tem no mesmo lugar:
- **Readiness subjetiva diária** (sono, humor, dores, energia, estresse) via `add-daily-readiness-checkin`
- **Aderência semanal** (planejado vs realizado)
- **TSB + forma** com calibração automática de zonas
- **Métricas de corrida** (Pa:HR, running dynamics em breve)

Nenhum concorrente (TrainingPeaks, Final Surge, Trenara) correlaciona **readiness subjetiva × TSB objetivo × aderência** num único lugar. O Menthoros já tem os 3 dados — mas não há tela que os cruze.

**Ideia:** Um "Readiness vs Reality Report" que mostre, para cada atleta: "você reportou PRONTO mas seu TSB estava −25 e sua aderência foi 40%". Isso é um sinal de overtraining que o coach demoraria semanas para perceber sozinho. Zero backend novo (dados já existem), frontend-only.

### 7.2 Coach onboarding friction

O coach hoje faz: criar assessoria → convidar atleta → atleta aceita → atleta loga treino manual ou upload .fit → coach vê dado → coach gera plano → coach revisa → coach aprova → atleta vê plano.

São **8 passos** antes do primeiro valor. Cada passo é uma oportunidade de abandono.

**Ideia:** Um "plano template" pré-gerado no momento do convite — o atleta recebe o convite E um plano inicial básico (baseado só em perfil: nível + dias disponíveis + objetivo). O coach revisa depois. Reduz a jornada de 8 para 3 passos (convite → atleta aceita → atleta já vê plano). Isso é o que o Freeletics faz com "plano inicial", mas com coach-in-the-loop.

### 7.3 Previsão de churn do atleta

Com os dados já coletados (readiness, aderência, streak, TSB), é possível construir um **score de risco de abandono** determinístico (sem IA) que alerte o coach: "Atleta X não loga treino há 5 dias, readiness caindo, streak quebrou — risco alto de abandono".

Isso não é `add-athlete-retention-loop-90d` (que é um sistema completo de intervenção) — é um sinal simples na fila de atenção, usando dados que já existem. XS, backend-only, uma query.
